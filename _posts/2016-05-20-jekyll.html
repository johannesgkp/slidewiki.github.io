Extending our developer website with a custom blog


As a catchy introduction for developers we created the SlideWiki GitHub page slidewiki.github.io. Because GitHub is a dream of a developer platform, we also want to use it to publish our inventions and hacks. Thus Jekyll is the best solution in order to work with GitHub pages and multiple developers who want to publish content.

The following will descripe how Jekyll is used, how to write your Jekyll blog locally and why we don't use jekyll-bootstrap.

For me Jekyll is the same for GitHub pages as SASS is for CSS. You write your content with help of the engine - e.g. modularity, reusability, linking - and it's parsed with the engine to your static content. So you define the default style for types, tags it and write pages which handles all the content and style with Liquid.

At the moment we have two pages, one as the entry point which the whole overview of what we think is interesting and the other which lists all blog entrys. These are written in HTML and include both the well know Bootstrap within JQuery and Font Awesome. The blog entrys are also written in HTML to use the same style as the two pages. We don't benefit from the features Jekyll can give us. Only when we migrate our style from bootstrap to the Jekyll environment we will change this. In order to add your blog entry just clone the repo: git clone git@github.com:slidewiki/slidewiki.github.io.git Add your new blog entry: cp _posts/2016-05-19-example-post.html _posts/YYYY-MM-DD-name.html Change the metadata and the content with your favourite editor. Preview it with docker: docker run --rm --label=jekyll --volume=$(pwd):/srv/jekyll --privileged=true -it -p 127.0.0.1:4000:4000 jekyll/jekyll If you are satisfied with it push it to the repo: git commit -am "added my blog entry" && git push
